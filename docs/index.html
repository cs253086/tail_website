TAIL OS documentation

TAIL OS is a microkernel real-time operating system written in Rust. It's built for robotics, autonomous vehicles, and other safety-critical systems where timing is a hard requirement, not a nice-to-have.

Version: v0.1.0 · Early preview

Overview

TAIL OS ships with:

  • A small, auditable microkernel that runs only scheduling, IPC, and memory management in privileged mode.
  • A deterministic EDF (Earliest-Deadline-First) scheduler with sub-microsecond IPC jitter.
  • Userspace drivers — UART and SD card today, more on the way.
  • A Rust async runtime tailored for no_std, with typed message channels and bounded queues.
  • A developer shell (tailos>) for inspecting tasks, tracing deadlines, and toggling drivers.

Installation

Prerequisites

  • Rust nightly with rust-src & llvm-tools-preview
  • QEMU (for emulation) or a Raspberry Pi 3B + SD card
  • A POSIX shell (macOS / Linux) or PowerShell (Windows)

One-line installer

macOS / Linux
curl --proto '=https' --tlsv1.2 -sSf https://tail-os.com/install.sh | sh

TAIL OS is currently a closed-source preview. The installer fetches signed prebuilt images; source access is available under evaluation agreement — contact us.

Quick start

Boot the prebuilt QEMU image:

shell
tailos pull --target qemu-aarch64
tailos run  --target qemu-aarch64

You'll land in the tailos> shell. Try:

tailos>
help
ps
trace control_loop --for 1s

Tasks & scheduling

Tasks in TAIL OS are async Rust functions annotated with #[tail_os::task]. Every task declares its priority, period, and optional deadline — the scheduler refuses to admit task sets it can't guarantee.

src/tasks.rs
#[tail_os::task(priority = 0, period = "1ms", deadline = "900us")]
async fn control_loop() {
    loop {
        let sample = imu::read().await;
        motor::apply(pid::step(sample));
        task::yield_now().await;
    }
}

IPC channels

Channels are typed, bounded, and lock-free. No unsafe required at the app level.

src/ipc.rs
use tail_os::ipc::channel;

let (tx, rx) = channel::<ImuSample>(16); // capacity = 16
tx.send(sample).await?;
let next = rx.recv().await?;

Drivers

All drivers live in userspace. Today v0.1 ships:

  • uart0 — Raspberry Pi 3B PL011 UART (115200 8N1)
  • sdhc — SD / MMC card host controller, read & write

Writing a driver is a crate — expose a capability, handle your IRQs, return messages. That's it.

The tailos> shell

A built-in REPL for introspection. Useful commands:

tailos>
help                 # list commands
ps                   # list tasks with CPU% & deadline stats
trace <task> --for 1s  # WCET / jitter / deadline misses
drv list             # installed drivers
meminfo              # kernel + userspace memory
reboot               # clean reboot

CLI reference

tailos --help
tailos <command> [options]

  run       Boot an image (QEMU or attached board)
  pull      Fetch a prebuilt image

FAQ

Is TAIL OS open source?

No — TAIL OS is currently a closed-source preview. Early access is shared for evaluation under agreement; contact us if you'd like to take it for a spin.

Which boards are supported?

Raspberry Pi 3B and QEMU virt (aarch64) today. Cortex-M and RISC-V targets are in progress.

Why a microkernel?

Smaller attack surface, drivers can crash without taking the kernel down, and it's far easier to reason about determinism when the kernel only does three things.
