TAIL OS documentation

TAIL OS is a microkernel real-time operating system written in Rust. It's built for robotics, autonomous vehicles, and other safety-critical systems where timing is a hard requirement, not a nice-to-have.

Version: v0.9.0 · Early preview

Overview

v0.9.0 ships with:

  • Microkernel — only scheduling, IPC, and memory run privileged. Everything else is userspace.
  • Real-time scheduler — deterministic, priority-based, with sub-microsecond IPC jitter.
  • Userspace drivers — UART and SD card (SDHC).
  • Rust async runtime — no_std, typed message channels, bounded queues.
  • Developer shell (tsh>) — inspect tasks, trace deadlines, toggle drivers.

Microkernel

TAIL OS follows a strict microkernel design: the kernel runs only three services in privileged mode — the scheduler, IPC message passing, and memory management. Everything else — drivers, protocol stacks, application tasks — runs as unprivileged userspace processes.

This boundary is a hard architectural rule, not a convention. A driver bug or crash stays contained in userspace and cannot corrupt kernel state. The kernel surface is small enough to audit in full.

Scheduler

The scheduler is built for real-time workloads. Tasks declare a priority and, optionally, a period and deadline. The scheduler enforces timing contracts at admission time — if a task set cannot be guaranteed to meet its deadlines, the system rejects it before starting.

Task execution is deterministic: no background daemons, no global GC, no unbounded critical sections. IPC jitter is kept sub-microsecond so that tight control loops get the timing they need.

IPC

Tasks communicate through typed, bounded message channels implemented in the kernel. The channel API is exposed as a Rust async interface — senders await when the buffer is full, receivers await when it is empty.

Channels are lock-free and require no unsafe at the application level. Because messages are typed at compile time, protocol mismatches are caught by the Rust compiler, not at runtime.

FAQ

Is TAIL OS open source?

No — TAIL OS is currently a closed-source preview. Early access is shared for evaluation under agreement; contact us if you'd like to take it for a spin.

Which boards are supported?

Raspberry Pi 3B and QEMU virt (aarch64) today. Cortex-M and RISC-V targets are in progress.

Why a microkernel?

Smaller attack surface, drivers can crash without taking the kernel down, and it's far easier to reason about determinism when the kernel only does three things.
