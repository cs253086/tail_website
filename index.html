Why TAIL OS

Built for the next generation of robotics.

A microkernel designed from first principles — deterministic, safe, and small enough to audit.

Microkernel core

A minimal, auditable kernel with userspace drivers. If it doesn't need to be privileged, it isn't.

Hard real-time

Deterministic EDF scheduler with sub-microsecond IPC jitter. Predictable by design, not by accident.

Memory-safe by construction

Written in safe Rust. No null pointers, no data races, no undefined behavior — enforced by the compiler.

Embedded & portable

Runs on Raspberry Pi 3B today, QEMU virt tomorrow, your custom board next. AArch64 first, more to come.

Safer IPC

Typed message channels with bounded queues. Share memory when you need to, copy when you should.

Predictable latency

No global GC, no background daemons, no surprises. Your control loop runs when it's scheduled — period.

Installation

Up and running in under 5 minutes.

Pick your host OS, grab the toolchain, flash it to a board or boot it in QEMU.

  1. Install prerequisites

    A Linux host (Ubuntu 22.04 / 24.04 tested) with QEMU for AArch64 and curl.

    Ubuntu / Debian
    sudo apt-get install -y qemu-system-aarch64 qemu-utils

  2. Run TAIL OS (one command)

    The launcher checks your QEMU install, caches the prebuilt images in ~/.cache/tailos/, and boots with the correct flags.

    shell
    curl -sSL https://raw.githubusercontent.com/cs253086/tailos/main/scripts/run_tailos_qemu.sh | bash

    When you see tsh>, the system is ready. Exit QEMU with Ctrl-A, then X.

  3. Manual launch (alternative)

    Prefer not to run a remote script? Grab the two images and boot QEMU yourself:

    shell
    wget https://tail-os.com/wp-content/uploads/2025/09/tail.rfs
wget https://tail-os.com/wp-content/uploads/2025/09/tail_disk.img

qemu-system-aarch64 \
    -M raspi3b \
    -kernel tail.rfs \
    -serial mon:stdio \
    -drive file=tail_disk.img,format=raw,if=sd

  1. What you need

    • Raspberry Pi 3B or 3B+
    • MicroSD card (8 GB or larger)
    • USB-to-TTL serial adapter (3.3 V) — FTDI, CP2102, etc.
    • 5 V / 2.5 A power supply (official RPi PSU recommended)
    • A Linux host machine for building and flashing

  2. Flash the SD image

    Grab the prebuilt tailos_sd.img (two-partition FAT32: boot + data) from the download page and write it to your SD card.

    Linux host
    # Flash to SD card (replace /dev/sdX with your SD card device)
sudo dd if=tailos_sd.img of=/dev/sdX bs=4M status=progress
sync

    Double-check /dev/sdX — picking the wrong device will overwrite the wrong disk.

  3. Wire the serial console

    Connect your USB-TTL adapter to the RPi3 GPIO header. TX and RX must be crossed.

    wiring
    USB-TTL Adapter          RPi3 GPIO Header
+-----------+            +----------------+
|  TX  o----+----------->| Pin 10 (RXD)  |  GPIO 15
|  RX  o<---+-----------o| Pin 8  (TXD)  |  GPIO 14
|  GND o----+----------->| Pin 6  (GND)  |
+-----------+            +----------------+

    Do not connect the VCC / 5 V pin from the adapter to the Pi — power the Pi from its own PSU.

  4. Power on & open a terminal

    Insert the SD card, apply power, then open a serial console at 115200 8N1:

    host
    screen /dev/ttyUSB0 115200
# or
minicom -D /dev/ttyUSB0 -b 115200

    When you see tsh>, TAIL OS is booted and ready on your Pi.
